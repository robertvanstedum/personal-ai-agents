## Why this is the priority

The Portuguese domain (Spec 1 — Auth) will touch the portal login
flow, session management, and route protection. Without automated
regression tests, a broken login won't be caught until Robert or
a daughter can't get in on a live production system.

The test suite is as important as the deploy pipeline. Both ship
together in this spec.

---

## Two deliverables

**1. Regression test suite** — automated tests covering all existing
domains. Run on every push. Block deploy if any fail. These protect
German, Curator, and Guild from being broken by Portuguese changes.

**2. Deploy pipeline** — push to main → images build → EC2 updates.
No manual steps for routine deploys. Defects filed automatically
as GitHub issues when tests fail.

---

## Goal

Push to main → regression tests run → if green, five Docker images
build → push to ECR → EC2 pulls and restarts → smoke tests run →
Telegram notification. Defects auto-filed as GitHub issues on failure.

No SSH keys. No manual steps. No browser terminal for routine deploys.

## Defect capture — automatic GitHub issues on test failure

When tests fail in CI, automatically open a GitHub issue:

```python
# scripts/file_defect.py
import argparse, os, requests

def file_issue(test_name, failure_message, sha, branch):
    token = os.environ['GITHUB_TOKEN']
    repo = 'robertvanstedum/personal-ai-agents'
    title = f"Test failure: {test_name} [{sha[:7]}]"
    body = (
        f"## Automated defect report\n\n"
        f"**Test:** `{test_name}`\n"
        f"**Branch:** {branch}\n"
        f"**Commit:** {sha}\n\n"
        f"**Failure:**\n```\n{failure_message}\n```\n\n"
        f"**Actions run:** "
        f"https://github.com/{repo}/actions\n\n"
        f"*Filed automatically by CI pipeline.*"
    )
    requests.post(
        f"https://api.github.com/repos/{repo}/issues",
        headers={'Authorization': f'token {token}'},
        json={'title': title, 'body': body, 'labels': ['defect', 'ci-failure']}
    )
```

GitHub Actions passes test results to this script. One issue
per failing test. Issues are labeled `defect` and `ci-failure`
for easy filtering.

**Rerun flow:**
1. Test fails → issue filed automatically
2. Robert or Claude Code fixes the issue
3. Push fix to main
4. Pipeline reruns — if green, issue can be closed
5. Robert approves deploy if all tests pass

## GitHub Actions Workflow

**File:** `.github/workflows/deploy.yml`

### Triggers

```yaml
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  workflow_dispatch:
    inputs:
      image_tag:
        description: 'Image tag to deploy (SHA or latest)'
        default: 'latest'
```

- Push to `main` → full pipeline
- PR to `main` → tests only, no deploy
- Manual trigger → deploy specific tag (rollback path)
- Other branches → nothing

### Job 1: test

```yaml
test:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-python@v5
      with:
        python-version: '3.11'
    - name: Install dependencies
      run: pip install -r requirements.test.txt
    - name: Run regression tests
      run: pytest tests/ -v --junit-xml=reports/junit.xml
    - name: Upload test report
      uses: actions/upload-artifact@v4
      with:
        name: test-report
        path: reports/junit.xml
      if: always()
    - name: File defects for failures
      if: failure()
      env:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        SHA: ${{ github.sha }}
        BRANCH: ${{ github.ref_name }}
      run: |
        pip install requests -q
        python3 scripts/file_defect.py \
          --junit reports/junit.xml \
          --sha $SHA \
          --branch $BRANCH
```

### Job 2: build-push (only on main, not PRs)

```yaml
build-push:
  needs: test
  if: github.ref == 'refs/heads/main' && github.event_name != 'pull_request'
  runs-on: ubuntu-latest
  outputs:
    image_tag: ${{ steps.tag.outputs.sha }}
  steps:
    - uses: actions/checkout@v4
    - name: Set image tag
      id: tag
      run: echo "sha=${GITHUB_SHA::7}" >> $GITHUB_OUTPUT
    - name: Configure AWS credentials
      uses: aws-actions/configure-aws-credentials@v4
      with:
        aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
        aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
        aws-region: us-east-1
    - name: Login to ECR
      uses: aws-actions/amazon-ecr-login@v2
    - name: Build and push all images
      run: |
        SHA=${{ steps.tag.outputs.sha }}
        REGISTRY=332704997792.dkr.ecr.us-east-1.amazonaws.com
        # All Dockerfiles are in docker/ directory; build context is repo root
        declare -A IMAGES=(
          ["portal"]="docker/Dockerfile.portal"
          ["curator"]="docker/Dockerfile.curator"
          ["mein-deutsch"]="docker/Dockerfile.german"
          ["system-bot"]="docker/Dockerfile.telegram"
          ["cos-bot"]="docker/Dockerfile.cos-bot"
        )
        for name in "${!IMAGES[@]}"; do
          dockerfile="${IMAGES[$name]}"
          docker build \
            -f $dockerfile \
            -t $REGISTRY/minimoi/$name:$SHA \
            -t $REGISTRY/minimoi/$name:latest \
            .
          docker push $REGISTRY/minimoi/$name:$SHA
          docker push $REGISTRY/minimoi/$name:latest
        done
```

### Job 3: deploy (EC2 via SSM — no SSH key)

```yaml
deploy:
  needs: build-push
  if: github.ref == 'refs/heads/main' && github.event_name != 'pull_request'
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - name: Configure AWS credentials
      uses: aws-actions/configure-aws-credentials@v4
      with:
        aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
        aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
        aws-region: us-east-1
    - name: Push docker-compose.prod.yml to EC2
      run: |
        COMPOSE=$(cat docker-compose.prod.yml | base64 -w 0)
        aws ssm send-command \
          --instance-ids ${{ secrets.EC2_INSTANCE_ID }} \
          --document-name "AWS-RunShellScript" \
          --parameters "commands=[
            'echo $COMPOSE | base64 -d > /opt/minimoi/docker-compose.prod.yml'
          ]" --output text
    - name: Deploy on EC2
      run: |
        aws ssm send-command \
          --instance-ids ${{ secrets.EC2_INSTANCE_ID }} \
          --document-name "AWS-RunShellScript" \
          --parameters "commands=[
            'cd /opt/minimoi',
            'aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin 332704997792.dkr.ecr.us-east-1.amazonaws.com',
            'docker-compose -f docker-compose.prod.yml pull',
            'docker-compose -f docker-compose.prod.yml up -d',
            'sleep 30',
            'curl -sf http://localhost:5001/health || exit 1',
            'curl -sf http://localhost:8766/health || exit 1',
            'curl -sf http://localhost:8767/health || exit 1'
          ]" --output text
```

### Job 4: notify (always runs)

```yaml
notify:
  needs: [test, build-push, deploy]
  if: always()
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - name: Send Telegram notification
      env:
        TELEGRAM_TOKEN: ${{ secrets.TELEGRAM_BOT_TOKEN }}
        TELEGRAM_CHAT_ID: ${{ secrets.TELEGRAM_CHAT_ID }}
        DEPLOY_STATUS: ${{ needs.deploy.result }}
        TEST_STATUS: ${{ needs.test.result }}
        SHA: ${{ github.sha }}
      run: |
        pip install requests -q
        python3 scripts/notify_deploy.py \
          --deploy-status $DEPLOY_STATUS \
          --test-status $TEST_STATUS \
          --sha ${SHA::7} \
          --branch ${{ github.ref_name }}
```

**Notification format:**

Success:
```
✅ mini-moi deployed
Branch: main | Commit: a1b2c3d
Tests: 12/12 passed
```

Failure (tests):
```
❌ Deploy blocked — tests failed
Branch: main | Commit: a1b2c3d
Failed: test_german_health, test_personas_load
2 GitHub issues filed automatically
```
