# journal/

Public narrative — sprint notes, build history, plans. Shows project evolution on GitHub.

**Committed and public.**

## Naming Convention

`YYYY-MM-DD-description.md` — e.g. `2026-03-21-repo-cleanup-release.md`

## What belongs here

- Sprint summaries
- Build phase notes (SPRINT_*, BUILD_WS*, PHASE_* files as they're graduated)
- Release notes
- Planning docs that are complete and worth publishing as part of the case study record

## What doesn't belong here

- Active working docs → `_working/`
- Private context or profile → `context/`
- Unfinished domains → `_NewDomains/`
