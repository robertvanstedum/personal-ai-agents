## Task 2 — HTML Reading Interface (web/generate.js)

A Node.js script that generates a static `web/index.html` from the library markdown files. Runs offline. No server. No external npm dependencies beyond Node.js built-ins.

**Run command:** `node web/generate.js` from project root

### What it reads
- `library/README.md` — master index table (all research items)
- `library/session-log.md` — last 10 session entries
- `library/reading-list.md` — books flagged for Robert
- `library/essays/` — list all `.md` files, read first line (title) and second non-empty line (summary)

### What it generates: web/index.html

Single file. Sections:

1. **Header** — project name, last generated timestamp, budget status (reads cumulative from session-log.md)

2. **Library** — searchable, filterable table of all research items
   - Search: keyword across title, summary, source fields (client-side, no backend)
   - Filter by: topic (dropdown), language (dropdown), type (dropdown: paper/book/article/essay/translation)
   - Each row: date, topic, source, language, type, summary, link (local path or URL)
   - Local paths render as `<code>` text (can't hyperlink local files reliably cross-browser)
   - URLs render as clickable links, open in new tab

3. **Essays** — agent-written syntheses
   - List from `library/essays/*.md`
   - Show: title (first line of file), date (from filename), 2-sentence summary (second non-empty paragraph), file path

4. **Reading List** — books flagged for Robert
   - Table from `library/reading-list.md`
   - Columns: title, author, why it matters, date added

5. **Recent Sessions** — last 10 rows from session-log.md
   - Show: date, session type, duration, cost, cumulative, notes

### Visual style
- Parchment aesthetic — matches Curator (Robert's existing system)
- Colors: `#f5f0e8` background (cream), `#2c1a0e` text (dark brown), `#8b6914` accents (muted amber)
- Font: system serif for body, monospace for paths/code/costs
- No external CDN, no Google Fonts — system fonts only for offline reliability
- Clean, readable on MacBook. Mobile-readable but not the priority.

### Constraints
- Pure HTML/CSS/JS output — works by opening `index.html` directly in browser (file://)
- No frameworks (no React, Vue, etc.)
- No npm dependencies — only Node.js built-ins (`fs`, `path`)
- Graceful handling of empty sections (show "No entries yet" rather than broken table)
- Regenerates cleanly every time — idempotent
